什麼是城市故障分析

我們把日常城市生活中反覆出現、容易造成不便或風險的問題，整理成可以追蹤的題組與熱點。

what it means

為什麼要用資料看城市問題

因為單一事件很容易被情緒放大，但資料可以幫我們看見：哪些問題反覆出現、集中在哪裡、屬於哪一種議題，接著才能討論改善順序與治理行動。

城市故障不等於城市失敗

這裡說的「故障」，比較像是城市運作中反覆出現的卡點。例如停車動線、路口安全、垃圾動線、市場周邊環境或學校周邊通行風險。

怎麼分類問題

目前先用容易理解的議題分類來整理城市問題，讓首頁的議題排行與熱點分析能夠對得起來。

issue categories
交通 道路 停車 / 人行 環境衛生 通學安全 市場商圈 公共安全 文化觀光

這些分類不是永久不變，而是 prototype dashboard 目前最容易讓一般人理解的題組。正式資料接入後，分類可以更細，也可以把問題之間的關聯看得更清楚。

怎麼看熱點

熱點不是單純看聲量，而是看資料在空間與議題上是否反覆集中，並且能不能轉成可追蹤的治理線索。

hotspot logic

熱點判讀原則

  • 看問題是否反覆出現在同一個區域。
  • 看它是否對交通、環境或通學安全造成實際影響。
  • 看是否能對應到具體主管機關與後續行動建議。
prototype hotspot score 不是正式行政評分。
它的作用是先讓使用者看懂：哪裡最值得優先觀察、哪一類城市問題現在最突出。

prototype data 的限制

這一點要講清楚，因為公開頁不能把 prototype data 說成正式資料。

honest limits

正式資料接入前，這些限制一直存在

目前網站仍是 prototype dashboard、非正式全量資料。這表示數字有展示價值，但不代表已完成正式接入、正式清洗與正式驗證。正式資料還在盤點與接入前整理階段，並維持 no live crawler、no source_url requests、no personal data、no private complaint full text、no auto publish，且 approved_for_crawling 不自動設定。

後續如何接正式資料

接正式資料不是把網址丟進程式就好，而是要先確認來源、欄位、風險與公開邊界。

formal data later

接資料前會做什麼

  • 確認是不是官方資料來源。
  • 確認資料格式與公開條件。
  • 確認欄位是否適合整理成正式資料表。
  • 確認不會碰個資、私人陳情全文或未授權資料。

正式資料接入後會帶來什麼

一旦正式資料來源穩定，首頁的議題排行、熱點地圖與行動建議就能更接近長期追蹤工具，而不是只停留在公開展示用的 prototype data。

方法論頁的工作，不是把系統包裝成很神秘，而是把資料怎麼看、怎麼分類、怎麼限制、怎麼接正式資料，全部講清楚。這樣一般人才能信任這是一個誠實的公開城市問題儀表板。
