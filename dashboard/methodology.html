什麼是城市故障分析

plain language

用工程方法看城市

城市問題不是只有單一事件，而是可能反覆出現在同一地點、同一類型、同一流程的系統性問題。城市故障分析就是把這些問題整理成可追蹤的資料。

從抱怨變成改善清單

交通、停車、道路、環境、通學安全等問題，若能整理成分類、熱點、趨勢與行動建議，就能從零散意見變成具體改善方向。

問題如何分類

issue taxonomy

交通停車道路路平人行安全環境衛生公共安全水利排水市場商圈通學安全文化觀光行政服務

分類不是為了貼標籤，而是為了讓相似問題可以被累積、比較與追蹤。

怎麼看熱點

hotspot logic

1. 看位置

同一地點若反覆出現交通、停車、環境或安全問題，就可能形成城市熱點。

2. 看問題強度

目前用 prototype score 表示熱點強度，正式版會再接入更多資料欄位與時間序列。

3. 看可執行行動

熱點不是只做排名，還要對應主管機關、改善方向與後續追蹤。

prototype data 的限制

data limitation

目前能做什麼

  • 展示儀表板樣貌。
  • 呈現原型 KPI、議題排行與熱點。
  • 說明後續要接哪些正式資料。

目前不能說什麼

  • 不能說這是正式全量資料。
  • 不能拿 prototype 當正式民意或完整統計。
  • 不能把未確認來源標成 approved_for_crawling。

後續如何接正式資料

next data

確認官方來源

先確認資料來源、欄位、更新頻率與限制。

建立資料欄位

把時間、地點、問題類型、主管機關與處理狀態整理成可分析欄位。

更新儀表板

讓首頁、地圖、總控台從 prototype data 逐步接入正式資料。

安全邊界：prototype dashboard、非正式全量資料、no live crawler、no source_url requests、no personal data、no private complaint full text、no auto publish、approved_for_crawling 不自動設定。
