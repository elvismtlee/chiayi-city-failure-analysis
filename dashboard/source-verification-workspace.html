資料源人工檢查工作台

在啟動任何爬蟲之前，我們先人工打開官方資料來源，確認它是不是官方資料、資料格式是什麼、有沒有個資風險、未來能不能整理成資料庫。

開始看資料源卡片 回到官方資料盤點 查看健康檢查

這個頁面是幹嘛的

它把原本太抽象的人工審核流程，縮成一個一般人也能照著做的工作台。你只要挑一筆官方資料來源，人工打開頁面，填完 8 個欄位，就能留下清楚的檢查結果。

  • 先看哪個官方資料來源
  • 再打開哪個 URL
  • 看完要填哪幾個欄位
  • 最後留下可以回填的檢查結果

今天真正要做什麼

只做人工檢查，不做任何程式抓取
1
打開一筆官方 URL 從下方卡片挑一筆官方資料來源，直接人工點開，確認這個網址真的是官方提供的頁面或資料集入口。
2
看頁面上有什麼資料 記下頁面標題、官方單位、資料格式，以及肉眼可以看到的主要欄位或資訊內容。
3
填 8 個檢查欄位 把檢查結果填成一致格式，留下下一步建議，並維持 crawler_execution_allowed = false。

今天會填的 8 個欄位

每一筆資料源只要先完成這 8 個欄位
source_title

頁面標題或資料集名稱，看完就照你眼前看到的名稱填。

source_owner

誰提供這份資料，例如嘉義市政府交通處、文化局、社會處。

source_url

你人工打開的網址，保留原始來源入口。

is_official_source

這是不是官方來源，先填 yes / no / unsure 都可以。

data_format

資料長什麼樣子，例如 CSV、HTML 頁面、JSON API 或 PDF。

visible_fields

頁面上目前看得到哪些欄位，例如停車場名稱、地址、更新日期。

privacy_risk

有沒有個資或私人陳情全文風險，可先填 low / medium / high。

recommended_next_action

這筆資料檢查完後建議做什麼，例如繼續人工盤點、等補欄位、暫不處理。

範例：嘉義市公有路外停車場資訊

下面這筆示範的重點，是讓使用者知道「看完頁面之後到底要留下哪些文字」，不是宣告這筆資料已經審核完成。

source_title嘉義市公有路外停車場資訊
source_owner嘉義市政府交通處
source_urlhttps://data.chiayi.gov.tw/opendata/dataset/metadata?oid=d206db33-3ae7-489e-b709-5555222fb767
is_official_sourceyes，因為網域是 data.chiayi.gov.tw，且頁面顯示官方單位。
data_formatCSV metadata page
visible_fields示意可寫：停車場名稱、地址、車位數、收費方式、更新日期。
privacy_risklow，因為看起來是公共停車設施資訊，不是個人資料。
recommended_next_action繼續人工確認欄位名稱，之後再評估是否適合整理成結構化資料。

這筆範例要達成的效果很簡單：看完官方頁面後，你可以用白話把判斷寫下來，而不是只留下看不懂的技術欄位名稱。

審完這一筆，會得到什麼結果

  • 知道這是不是官方來源，而不是只是看起來像官方。
  • 知道資料格式是 CSV、HTML 頁面、JSON API 還是其他格式。
  • 知道目前肉眼看得到哪些欄位，未來有沒有機會整理成資料表。
  • 知道這筆資料的個資或隱私風險高不高。
  • 知道下一步是繼續人工整理，還是先停下來不要碰。

為什麼目前還不能開 crawler

因為現在還在資料源人工檢查階段。很多官方頁面只是入口頁、說明頁或服務頁，還沒有人工確認它們適不適合整理成資料。只要還沒完成這一層檢查，就不能把任何資料標成 approved_for_crawling，也不能讓 crawler_execution_allowed 變成 true。

安全邊界

這些規則現在都還是硬邊界
目前不啟動 live crawler no source_url requests 不抓私人陳情全文 不抓個資 不自動發布 不自動寄信 approved_for_crawling 不可自動設定 crawler_execution_allowed = false engineering_review_allowed = false
