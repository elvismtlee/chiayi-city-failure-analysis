# 資料治理與公開說明 v1

本文件定義嘉義城市故障分析資料庫的資料治理原則。

---

## 基本原則

1. 以官方公開資料為優先。
2. 所有資料應保留來源 URL、抓取時間與原始欄位。
3. 不公開個人資料與可識別個資。
4. 不把資料不足解讀為沒有問題。
5. 不把單一資料源解讀為完整民意。
6. AI 摘要必須能追溯原始資料。
7. 原型資料必須明確標示 prototype / sample / mock。

---

## 資料類型

### 官方公開資料

例如：

- 嘉義市議會會議紀錄
- 嘉義市議員質詢影音 metadata
- 政府開放資料
- 市府公開統計

### 使用者回報資料

例如：

- 城市故障回報
- 地方議題意見徵集
- 西區意見牆

使用者回報資料需注意：

1. 不公開姓名、電話、email。
2. 僅公開議題分類、地點、摘要與統計結果。
3. 若要公開個案內容，需去識別化。

---

## 不應使用的說法

避免使用：

- 民調
- 支持度調查
- 投票意向
- 民意代表評分
- 官方效率排名定論

建議使用：

- 城市議題觀察
- 地方生活議題資料
- 公開資料分析
- 城市故障追蹤
- 議題關注度
- 局處案件負荷觀察

---

## AI 摘要限制

AI 摘要只能做：

- 整理
- 分類
- 摘要
- 趨勢提示
- 建議追蹤方向

AI 摘要不應做：

- 無來源指控
- 人身攻擊
- 絕對化結論
- 錯誤歸因
- 將資料不足視為事實

---

## 對外公開聲明建議

可放在 dashboard footer 或資料來源頁：

```text
本平台資料來自公開資訊、官方資料與原型資料整理。分析結果為城市治理與地方議題研究用途，不代表完整民意調查，也不作為個人評價結論。所有資料仍需回到原始來源查證。
```

## 資料更新狀態標示

每個 JSON 應包含：

```json
{
  "updated_at": "2026-05-21 09:00:00",
  "source": "prototype",
  "data_status": "mock"
}
```

正式資料可使用：

```text
raw
processed
verified
published
```
