# 資料發布政策 v1

本文件定義哪些資料可以公開、哪些資料不可公開，以及公開前需要經過哪些處理。

---

## 一、資料狀態分級

| 狀態 | 說明 | 是否可公開 |
|---|---|---|
| raw | 原始抓取資料 | 不一定，需檢查 |
| processed | 已處理資料 | 可內部使用 |
| deidentified | 已去識別化資料 | 可考慮公開 |
| verified | 已人工檢查資料 | 可公開 |
| published | 已發布資料 | 可公開 |
| mock | 原型假資料 | 可公開，但必須標示 |
| sample | 樣本資料 | 可公開，但必須標示 |

---

## 二、可公開資料

可公開：

1. 官方公開 metadata。
2. 已公開的政府統計資料。
3. 不含個資的開放資料。
4. 已去識別化的統計摘要。
5. 已人工檢查的分類資料。
6. prototype / mock data，但必須清楚標示。

---

## 三、不可公開資料

不可公開：

1. 姓名。
2. 電話。
3. email。
4. LINE ID。
5. 身分證字號。
6. 車牌。
7. 完整住家地址。
8. 可識別個人的具體描述。
9. 未確認授權的非公開資料。
10. 需要登入、驗證碼或非公開權限才能取得的資料。

---

## 四、公開前處理流程

```text
raw data
→ 欄位盤點
→ 個資檢查
→ 地址降精度
→ 議題分類
→ 地點標準化
→ JSON validation
→ 人工抽查
→ published
```

---

## 五、公開頁面必備資訊

每個公開資料頁應盡可能標示：

- 資料來源
- 更新時間
- 資料狀態
- 資料限制
- 是否為 prototype / mock data
- 原始來源連結

---

## 六、AI 產生內容發布規則

AI 摘要或週報可公開前，需符合：

1. 不含無來源指控。
2. 不含人身攻擊。
3. 不把樣本資料說成完整民意。
4. 不使用民調、支持度調查等字眼。
5. 有資料限制聲明。
6. 重要內容可追溯資料來源。

---

## 七、使用者回報資料發布規則

使用者回報資料只能公開：

- 議題分類
- 區域統計
- 模糊地點
- 去識別化摘要
- 熱點統計

不可公開：

- 原始全文
- 個人聯絡資料
- 完整地址
- 可能辨識個人的描述

---

## 八、撤下資料規則

若發現資料可能含個資、錯誤或授權疑慮：

1. 立即從公開 JSON 或頁面移除。
2. 保留內部處理紀錄。
3. 修正後重新產生 processed data。
4. 重新跑 JSON validation。
5. 再重新發布。

---

## 九、對外聲明

```text
本平台公開資料以官方公開資訊、開放資料、去識別化資料與原型資料整理為主。若資料涉及個人資訊、授權不明或尚未完成檢查，將不進行公開展示。
```
