# 個資去識別化規格 v1

本文件定義城市故障回報、1999 / 陳情資料與地方議題意見徵集資料在公開前的去識別化原則。

---

## 核心原則

1. 不公開姓名。
2. 不公開電話。
3. 不公開 email。
4. 不公開完整住家地址。
5. 不公開可識別個人的具體描述。
6. 對外呈現以統計、分類、摘要、熱點為主。

---

## 需要移除或遮蔽的欄位

| 欄位 | 處理方式 |
|---|---|
| name | 移除 |
| phone | 移除 |
| email | 移除 |
| line_id | 移除 |
| national_id | 移除 |
| exact_address | 降精度或移除 |
| free_text_personal_info | 人工或 AI 協助檢查後遮蔽 |

---

## 地址降精度規則

| 原始資訊 | 對外顯示 |
|---|---|
| 嘉義市西區某路 100 號 | 嘉義市西區某路段 |
| 住家門口 | 某路段 / 某里 |
| 私人公司地址 | 若非公開場所，降為路段 |
| 學校 / 公園 / 市場 | 可保留地標名稱 |
| 路口 | 可保留路口 |

---

## 文字描述去識別化

原文：

```text
我住在某路 100 號 3 樓，隔壁住戶每天晚上製造噪音。
```

公開摘要：

```text
民眾反映某路段夜間噪音問題，建議列為環境噪音議題追蹤。
```

```json
  "data_status": "deidentified"

```

---

## 不應公開的內容

不可公開：

- 具體門牌
- 個人姓名
- 電話
- email
- LINE ID
- 身分證字號
- 車牌號碼
- 醫療資訊
- 家庭成員資訊
- 可辨識特定個人的攻擊性描述

---

## AI 處理注意事項

AI 可協助：

- 摘要
- 移除個資
- 降低地址精度
- 分類議題

AI 不可自動公開：

- 未人工檢查的敏感文字
- 可能含個資的原始描述
- 涉及具體個人的指控

---

## 發布前檢查

每批公開資料前需檢查：

- [ ] 是否含姓名
- [ ] 是否含電話
- [ ] 是否含 email
- [ ] 是否含完整地址
- [ ] 是否含車牌
- [ ] 是否含醫療或家庭資訊
- [ ] 是否含可識別個人的敘述
- [ ] 是否已標示資料來源與限制

