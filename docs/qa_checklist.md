# QA 檢查清單 v1

本文件用於檢查嘉義城市故障分析資料站的內容品質、資料品質與前端品質。

---

## 1. 內容 QA

- [ ] 所有繁體中文正確
- [ ] 嘉義、嘉義市、西區等地名正確
- [ ] 不使用簡體字
- [ ] 不使用容易誤導的政治民調語言
- [ ] 不使用攻擊性語句
- [ ] 不把原型資料說成正式資料
- [ ] 所有頁面都有清楚定位

---

## 2. 資料 QA

- [ ] JSON 格式正確
- [ ] CSV 欄位符合 schema
- [ ] raw data 不覆蓋 processed data
- [ ] mock data 標示清楚
- [ ] 每個資料集有 updated_at 或 latest_update
- [ ] source_url 存在且可追蹤
- [ ] 個資未被公開

---

## 3. 前端 QA

- [ ] 首頁 KPI 正常顯示
- [ ] 熱點表正常顯示
- [ ] 議題排行正常顯示
- [ ] 城市洞察資料正常顯示
- [ ] 資料來源表格正常顯示
- [ ] 週報列表正常顯示
- [ ] 手機版可閱讀
- [ ] 按鈕和連結可點擊

---

## 4. 無障礙與可讀性

- [ ] 按鈕文字清楚
- [ ] 文字與背景對比足夠
- [ ] 表格在手機版可橫向閱讀或重新排版
- [ ] iframe 嵌入時高度足夠
- [ ] 圖表不要只靠顏色辨識

---

## 5. GitHub / Codex QA

- [ ] 每個 PR 任務單一明確
- [ ] PR description 包含測試結果
- [ ] GitHub Actions 通過
- [ ] Codex 沒有改到無關檔案
- [ ] 沒有把 credentials 放入 repo
- [ ] 新 branch 從最新 main 開出

---

## 6. 發布 QA

- [ ] GitHub Pages 部署成功
- [ ] 發布後清除瀏覽器 cache 再測
- [ ] 桌機 Chrome 測試
- [ ] 手機 Chrome / Safari 測試
- [ ] WordPress iframe 測試
- [ ] chiayi2026.com 導流按鈕測試

---

## 7. 每週維護 QA

- [ ] 檢查 crawler 是否成功
- [ ] 檢查資料筆數是否異常下降
- [ ] 檢查週報是否產生
- [ ] 檢查 AI 摘要是否有過度推論
- [ ] 檢查資料來源是否更新
