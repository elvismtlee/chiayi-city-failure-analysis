# 嘉義城市故障分析中心｜網站架構 v1

本文件定義 GitHub Pages 資料站的頁面層級、使用者路徑與未來擴充方向。

---

## 網站定位

「嘉義城市故障分析中心」是嘉義市 12 年城市故障分析資料庫的公開展示端。

它不是競選主官網，而是資料型副站，負責呈現：

- 城市問題總覽
- 生活議題熱點
- AI 城市摘要
- 趨勢分析
- 局處負荷觀察
- 議員議題分析
- 城市故障分數

---

## 第一層：首頁

```text
dashboard/index.html
```

公開網址：

```text
https://elvismtlee.github.io/chiayi-city-failure-analysis/
```

定位：

```text
城市儀表板首頁 / 戰情總覽入口
```

首頁應回答：

1. 目前城市最重要的問題是什麼？
2. 哪些熱點最值得注意？
3. 哪些議題最高頻？
4. 資料目前更新到哪裡？
5. 下一步要去哪裡看深入分析？

首頁主要區塊：

- Hero / 專案說明
- KPI 總覽
- 西區城市故障熱點圖
- 議題排行
- 城市故障追蹤表
- 前往城市洞察分析
- 資料工程進度

---

## 頁面導覽原則

1. 每一頁都應有頂部導覽列。
2. 首頁應能導到所有主要頁面。
3. 第二層頁面應能明確返回首頁。
4. 文字必須清楚標示目前頁面定位。
5. 不使用民調、支持度調查等容易誤解的用語。
6. 原型資料必須標示為 prototype / mock data。

## 與 chiayi2026.com 的關係

```text
chiayi2026.com
```

是主官網，負責：

- 候選人介紹
- 政見主張
- 活動消息
- 加入我們
- 城市回報表單

```text
GitHub Pages dashboard
```

是資料副站，負責：

- 資料儀表板
- AI 分析
- 城市故障追蹤
- 公開資料展示

建議主官網用按鈕導流到資料站，正式穩定後再考慮 iframe 或子網域。
