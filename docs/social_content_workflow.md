# 城市故障資料轉社群內容流程 v1

本文件定義如何把 dashboard、週報與現場行動資料轉成社群內容。

---

## 目標

讓城市故障資料可以轉成市民聽得懂、願意分享、又不踩法律與倫理風險的內容。

---

## 資料來源

可用來源：

- dashboard_summary.json
- issue_ranking.json
- hotspots.json
- issue_trends.json
- ai_issue_summary.json
- weekly report
- field action notes

---

## 一題多用格式

每個議題可以轉成：

```text
1. FB 長文
2. Threads 短文
3. LINE 群組版
4. 30 秒短影音口播
5. 海報標語
6. 里鄰拜訪問句
7. 週報段落
```

---

## 輸入格式

```json
{
  "issue": "市場周邊停車與人行安全",
  "location": "嘉義市西區市場周邊",
  "summary": "尖峰時段臨停與卸貨動線影響行人通行。",
  "data_status": "prototype",
  "recommended_action": "盤點臨停熱點、卸貨時間與行人穿越動線。"
}
```

## 避雷規則

不可寫：

- 這證明大家都不滿
- 某局處失職
- 某議員不關心
- 民調顯示
- 支持度下降

應改寫：

- 資料中可觀察到
- 可列為後續追蹤
- 值得進一步確認
- 可作為城市治理參考
- 目前仍需更多資料佐證
