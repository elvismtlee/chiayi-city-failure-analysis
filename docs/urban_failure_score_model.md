# 城市故障分數模型 v1

本文件定義城市故障分數的計算概念。

## 目的

把不同來源的資料轉成可比較、可排序、可視覺化的城市故障指標。

---

## 基本分數

```text
Urban Failure Score =
案件量分數 + 重複率分數 + 處理時間分數 + 質詢落差分數 + 趨勢升高分數
```

---

## 指標定義

| 指標 | 說明 | 權重建議 |
|---|---|---|
| case_volume_score | 案件量 | 30% |
| repeat_score | 同地點 / 同議題重複率 | 20% |
| processing_delay_score | 處理天數 | 20% |
| question_gap_score | 市民反映多但質詢少 | 20% |
| trend_score | 近期是否升高 | 10% |

---

## 分數區間

| 分數 | 等級 | 說明 |
|---|---|---|
| 0 - 39 | 低 | 觀察即可 |
| 40 - 69 | 中 | 需要追蹤 |
| 70 - 84 | 高 | 應納入優先議題 |
| 85 - 100 | 極高 | 應列為城市故障熱點 |

---

## 輸出資料

```text
dashboard/data/urban_failure_scores.json
```

## JSON 範例

```json

  {

    "target_name": "文化路夜市",

    "issue": "停車 / 人行",
    "score": 92,
    "level": "極高",
    "main_reason": "案件量高、重複率高、近期趨勢升高",
    "recommended_action": "夜間停車與人行動線專案"
  }
]
```

## 應用

- 熱點排名
- 里別排名
- 路段排名
- 議題排名
- 城市週報
- AI 摘要
- 政策優先順序

---

## 注意事項

本分數是治理輔助工具，不是絕對評價。所有結果需附資料來源與更新時間。
